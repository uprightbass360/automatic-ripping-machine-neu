Three Approaches

How to add API key validation + permissions health to the setup wizard and settings

A

Minimal - Backend Only + Wizard Step

ARM-neu: Add POST /system/preflight endpoint (runs OMDB, TMDB, TVDB, MakeMKV checks + path ownership with UID:GID). Add GET /metadata/test-key?provider=tvdb for the missing TVDB test.

UI: Add a 4th wizard step "Readiness Check" that calls preflight and shows pass/fail cards. No changes to Settings page (it already has individual test buttons).

Scope: ~2 files backend, ~2 files frontend. Smallest change.

B

Wizard Step + Settings Health Panel

ARM-neu: Same preflight endpoint as A. Extend /system/paths to include UID:GID ownership info.

UI: Add wizard step like A. Also add a "System Health" card to the Settings page that shows path ownership, key status, and MakeMKV key validity. Always accessible, not just first-run.

Scope: ~3 files backend, ~4 files frontend. Medium change.

C

Wizard Step + Dashboard Health Banner

ARM-neu: Same preflight endpoint. Extend paths with UID:GID.

UI: Add wizard step. Add a persistent health banner on the Dashboard (not Settings) that shows warnings when keys are missing, paths aren't writable, or UID mismatches detected. Dismissable but re-checks periodically.

Scope: ~3 files backend, ~5 files frontend. Largest change but best ongoing UX.

Recommendation: B

The wizard step catches problems at first-run. The Settings health panel lets users re-run checks after changing config. A dashboard banner (C) is nice but adds ongoing polling complexity and visual noise for a problem that mostly matters at setup time.

The transcoder checks are shown only when a transcoder URL is configured - gracefully absent when transcoder is not deployed.
